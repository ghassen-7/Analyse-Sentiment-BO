Contexte & Enjeux

Le Défi du Langage Web

Les commentaires YouTube présentent des défis NLP spécifiques : argot, abréviations, emojis, fautes d'orthographe et structures grammaticales non conventionnelles ("C tro b1" vs "C'est très bien").

L'Approche Hybride

Combinaison de Transfer Learning (RoBERTa pour la labellisation de haute qualité) et de Machine Learning Supervisé (Léger et rapide pour la production).

1. Data Engineering

Construction d'un Ground Truth fiable via IA SOTA.

2. Modélisation

Entraînement et benchmark rigoureux.

3. Industrialisation

Interface utilisateur pour exploitation réelle.

Préparation des Données

Source & Défi

Extraction de 3 242 commentaires bruts via l'API YouTube Data sur des vidéos produits.

Problème : Données "Sauvages" non labellisées. Impossible d'entraîner un classifieur supervisé sans "Vérité Terrain".

Solution : AI-Labelling

Génération automatique des labels (Positif/Négatif/Neutre) via un modèle Transformer pré-entraîné sur 198M de tweets, capable de comprendre l'argot et les emojis.

Stratégie d'Équilibrage

Pourquoi équilibrer ?

Un dataset déséquilibré biaise fortement le modèle vers la classe majoritaire. Sans correction, l'algorithme maximiserait son accuracy en prédisant toujours "Positif", ignorant les signaux négatifs critiques pour notre analyse.

Techniques Appliquées

Random Undersampling

Réduction aléatoire contrôlée de la classe majoritaire (positifs) pour s'aligner sur le seuil cible.

Récupération des Neutres

Conversion d'avis neutres pour enrichir la classe minoritaire.

Victoire du LinearSVC

Les données textuelles vectorisées (TF-IDF) créent des espaces de très haute dimension souvent linéairement séparables. Les SVM linéaires excellent dans ce contexte, maximisant la marge entre positifs et négatifs là où les arbres peinent à trouver des seuils simples.

Échec du Decision Tree

Avec un Recall de seulement 53%, l'arbre unique souffre de sur-apprentissage (overfitting). Il capture bien les "règles exactes" du set d'entraînement mais échoue à généraliser sur les subtilités du langage naturel (ironie, contexte).

Pourquoi LinearSVC ?

Les vecteurs TF-IDF sont de haute dimension (5000+ features) et creux (sparse).

Les SVM Linéaires excellent à trouver des hyperplans séparateurs dans ces espaces complexes, là où les arbres peinent à créer des règles binaires efficaces sans profondeur excessive (overfitting).

Analyse des Erreurs (16.5%)

  • Sarcasme : "Bravo pour l'arnaque" → Classé Positif (Mot "Bravo").
  • Négation Complexe : "Pas mal du tout" → Parfois ambigu pour les n-grammes simples.
  • Contexte Manquant : Références externes non capturées par le texte seul.
